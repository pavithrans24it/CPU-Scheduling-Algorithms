Priority Scheduling

Algorithm Explanation

Priority Scheduling executes processes based on their priority values. Each process is assigned a priority number, and the process with the highest priority is executed first.

Key Characteristics:

  • Processes are assigned priorities (lower number = higher priority)
  • Can be preemptive or non-preemptive
  • This implementation uses non-preemptive priority
  • Provides flexibility in scheduling
  • Can cause starvation of low-priority processes

Priority Scheme:

  • Lower priority number = Higher priority (e.g., 1 is highest priority)
  • If two processes have the same priority, FCFS is applied

Advantages:

  • Flexible scheduling based on importance
  • Real-time systems compatibility

Disadvantages:

  • Starvation of low-priority processes
  • Context switching overhead
