Welcome to CPU Scheduling Algorithms

This interactive educational website demonstrates various CPU scheduling algorithms used in operating systems. Learn how different scheduling strategies allocate CPU time to processes, and visualize how each algorithm works with interactive examples.

Explore the algorithms below to understand their working principles, input your own process details, and see real-time Gantt charts showing the execution sequence.

First Come First Serve (FCFS)

The simplest scheduling algorithm where processes are executed in the order they arrive. Non-preemptive and easy to implement, but can lead to poor average waiting time.

Use Cases: Batch processing systems, simple scheduling needs

Explore FCFS →

Shortest Job First (SJF)

Executes the process with the smallest burst time next. Minimizes average waiting time and is optimal, but requires knowing burst times in advance.

Use Cases: Batch systems, offline scheduling

Explore SJF →

Priority Scheduling

Processes are assigned priorities, and the highest priority process runs next. Flexible scheduling based on importance, but can starve low-priority processes.

Use Cases: Real-time systems, mixed workloads

Explore Priority →

Round Robin (RR)

Each process gets a fixed time quantum. Preemptive scheduling that provides fair CPU allocation and low response time for interactive systems.

Use Cases: Time-sharing systems, interactive applications

About This Website

This educational platform provides interactive demonstrations of CPU scheduling algorithms. Each algorithm page includes:

  • Algorithm Explanation: Detailed description of how the algorithm works
  • Key Characteristics: Important properties and features
  • Interactive Input: Enter your own process details
  • Instant Calculations: See waiting times and turnaround times
  • Gantt Chart: Visual representation of process execution
  • Statistics: Average metrics for performance analysis

How to Use

  1. Select an algorithm from the navigation menu
  2. Enter the number of processes
  3. Fill in arrival times and burst times (and priority if applicable)
  4. For Round Robin, specify the time quantum
  5. Click "Calculate" to see the results
  6. View the Gantt chart for visual representation
  7. Compare statistics to understand algorithm efficiency

Key Terms

  • Arrival Time: When the process enters the ready queue
  • Burst Time: How long the process needs CPU time
  • Waiting Time: Time spent in the ready queue waiting for CPU
  • Turnaround Time: Total time from arrival to completion
  • Priority: Relative importance of a process (lower number = higher priority)
  • Time Quantum: Fixed time slice for each process in Round Robin
